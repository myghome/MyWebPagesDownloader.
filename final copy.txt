Improvements I'd still make
1. Add Configuration objects

Instead of reading configuration directly everywhere.

Add:

Infrastructure/
└── Configuration/
    ├── DownloaderOptions.cs
    ├── RetryOptions.cs
    └── StorageOptions.cs

Use:

IOptions<DownloaderOptions>

throughout the app.

2. Add DownloadResult

You removed it completely.

I would bring it back.

Core/
└── Models/
    └── DownloadResult.cs

Example:

public sealed record DownloadResult(
    bool Success,
    HttpStatusCode? StatusCode,
    long? ContentLength,
    string? ErrorMessage,
    TimeSpan Duration);

Workers can return this cleanly.

3. Add Correlation IDs

Production systems benefit greatly.

CorrelationId

inside:

DownloadRequest

Useful for:

tracing
logging
diagnostics
4. Add Hosted Services

Instead of:

Program.cs

running everything manually.

I'd use:

BackgroundService

Example:

Application/
└── HostedServices/
    └── DownloaderHostedService.cs

Cleaner startup and shutdown.

5. Add Repository Interface to Core

Currently:

Infrastructure/
└── Persistence/
    └── IDownloadRepository.cs

I'd move the interface to:

Core/
└── Contracts/
    └── IDownloadRepository.cs

Reason:

Application should depend on abstractions.

Infrastructure implements them.

Classic Dependency Inversion Principle.

One thing I'd definitely change

Current:

Persistence/
├── IDownloadRepository.cs
├── DownloadRepository.cs

Recommended:

Core/
└── Contracts/
    └── IDownloadRepository.cs

Infrastructure/
└── Persistence/
    └── SqliteDownloadRepository.cs

This is more aligned with Clean Architecture.

One feature still missing

A downloader eventually needs:

Resume Incomplete Downloads

Add:

Application/
└── Services/
    └── RecoveryService.cs

Startup flow:

App starts
↓
Query SQLite
↓
Find Pending/Running records
↓
Requeue them
↓
Resume work

That's a very real-world feature.
