A few small refinements
These are refinements, not problems.
1. Consider moving options out of Infrastructure
You currently have:
Infrastructure/ └── Configuration/
That's acceptable.
However, options are often treated as application-level concerns.
Alternative:
Application/ └── Options/     ├── DownloaderOptions     ├── RetryOptions     └── StorageOptions
Either approach is fine.
I would not change it unless you want strict Clean Architecture purity.

2. Add IDownloadQueue abstraction (optional)
Today:
ChannelOrchestrator
owns the queue.
You could introduce:
Core/ └── Contracts/     └── IDownloadQueue
Example:
ValueTask EnqueueAsync(    DownloadRequest request,    CancellationToken cancellationToken);IAsyncEnumerable<DownloadRequest> ReadAllAsync(    CancellationToken cancellationToken);
Then:
ChannelDownloadQueue
becomes one implementation.
This would make future replacement easier.
Not necessary, but elegant.

3. Add DownloadHistory queries (optional)
Eventually you'll want:
Last 100 downloadsFailed downloadsAverage duration
Consider:
IDownloadRepository
supporting query methods:
Task<IReadOnlyList<DownloadRecord>> GetFailedAsync(...)Task<IReadOnlyList<DownloadRecord>> GetRecentAsync(...)
Very useful for diagnostics.

4. Add schema versioning
Today:
DatabaseInitializer
creates tables.
Tomorrow:
v1↓v2 adds column↓v3 adds index
Consider a simple:
SchemaVersion
table.
No need for EF migrations.
Just basic version tracking.

5. Add duplicate URL strategy
Current validator says:
No duplicates
Define the rule explicitly.
Examples:
Option A
Duplicate URL rejected.
Option B
Duplicate URL allowed after N minutes.
Option C
Duplicate URL allowed if previous attempt failed.
Real systems usually need a clear rule.

One thing I'd definitely add
Correlation-aware logging
You already have:
CorrelationId
inside:
DownloadRequest
Use it everywhere.
Example logs:
[CorrelationId=abc123]Download started[CorrelationId=abc123]HTTP 200 received[CorrelationId=abc123]Content saved
This becomes very powerful once OpenTelemetry is enabled.
